# Руководство по разработке

Данный документ описывает, как запускать, тестировать и разрабатывать проект **tuxback** локально.

TuxBack — это CLI‑приложение на Python, которое предоставляет функции резервного копирования, восстановления данных и планирования задач для Linux‑систем.

---

# Требования

Перед началом работы с проектом убедитесь, что установлены следующие инструменты:

- Python **3.10+**
- Git
- Linux или macOS
- Docker (необязательно, используется для сборки контейнера)

---

# Клонирование репозитория

Склонируйте репозиторий проекта:

```bash
git clone https://github.com/de-frogg/tuxback
cd tuxback
```

# Локальный запуск CLI

CLI можно запускать напрямую без установки программы в систему.

Пример:

```bash
python3 cli.py --help
```

Создание тестового бэкапа:

```bash
python3 cli.py backup test_data
```

Просмотр списка резервных копий:

```bash
python3 cli.py list
```

Восстановление архива:

```bash
python3 cli.py restore <archive> ./restore_dir
```

---

# Тестирование планировщика

Логику планировщика можно протестировать вручную:

```bash
python3 cli.py run-scheduler
```

Эта команда имитирует выполнение задач резервного копирования по расписанию.

---

# Docker

Docker используется для проверки того, что приложение корректно собирается и работает в изолированной среде.

Сборка Docker‑образа:

```bash
docker build -t tuxback .
```

Запуск контейнера:

```bash
docker run --rm tuxback
```

Docker помогает гарантировать, что проект будет работать одинаково на разных системах.

---

# CI/CD

В проекте используется **GitHub Actions**.

CI‑pipeline автоматически выполняет:

- установку зависимостей
- проверку работы CLI
- сборку Docker‑образа

Это гарантирует, что после каждого коммита проект корректно собирается и запускается.

# Логирование

Логи приложения записываются в файл:

```
tuxback.log
```

Логи содержат информацию о:

- создании резервных копий
- операциях восстановления
- выполнении задач планировщика
- возникших ошибках

Пример записи в журнале:

```
2026-03-15 10:45:21 | INFO | cli | CLI command started: backup
```
