# Установка TuxBack

Данный документ описывает процесс установки и первоначальной настройки утилиты **TuxBack**.

TuxBack — это консольное приложение для Linux, предназначенное для создания, хранения и восстановления резервных копий директорий. Утилита также поддерживает автоматическое выполнение резервного копирования по расписанию.

---

# Системные требования

Перед установкой убедитесь, что в системе присутствуют следующие компоненты:

- операционная система Linux
- Python версии **3.10 или выше**
- Git
- systemd (используется для автоматического запуска задач резервного копирования)

В большинстве современных Linux‑дистрибутивов эти инструменты уже установлены.

---

# Получение исходного кода

Для установки необходимо скачать проект из репозитория GitHub.

Выполните следующие команды:

```bash
git clone https://github.com/de-frogg/tuxback
cd tuxback
```

После выполнения этих команд вы окажетесь в директории проекта.

---

# Подготовка установочных файлов

Перед установкой необходимо сделать исполняемыми скрипты запуска и установки.

Выполните команду:

```bash
chmod +x tuxback install.sh uninstall.sh
```

Эта команда позволяет запускать данные файлы как исполняемые программы.

---

# Установка программы

Для установки TuxBack выполните команду:

```bash
./install.sh
```

Во время установки выполняются следующие действия:

- копирование файлов проекта в директорию **/opt/tuxback**
- создание глобальной команды **tuxback**
- установка **systemd‑сервиса** для планировщика
- установка и активация **systemd‑таймера**

После завершения установки утилиту можно запускать из любой директории системы:

```bash
tuxback
```

---

# Проверка установки

Чтобы убедиться, что установка прошла успешно, выполните:

```bash
tuxback --help
```

Проверить версию программы можно командой:

```bash
tuxback --version
```

Также можно проверить текущее состояние системы резервного копирования:

```bash
tuxback status
```

Пример вывода:

```
TuxBack status:
- Version: 1.0.0
- Total backups: 0
- Total schedules: 0
- Enabled schedules: 0
```

---

# Проверка работы планировщика

Во время установки автоматически создаётся **systemd‑таймер**, который периодически запускает планировщик резервного копирования.

Проверить состояние таймера можно командой:

```bash
systemctl status tuxback-scheduler.timer
```

Посмотреть список всех активных таймеров:

```bash
systemctl list-timers
```

Просмотреть последние записи журнала выполнения задач:

```bash
journalctl -u tuxback-scheduler.service -n 20 --no-pager
```

---

# Удаление программы

Чтобы полностью удалить TuxBack из системы, выполните:

```bash
/opt/tuxback/uninstall.sh
```

Скрипт удаления выполнит следующие действия:

- остановит и отключит systemd‑таймер
- удалит systemd‑сервис
- удалит глобальную команду `tuxback`
- удалит директорию установки `/opt/tuxback`

---

# Очистка кеша команд оболочки

После удаления программа может оставаться в кеше оболочки.

Если команда `tuxback` всё ещё определяется системой, выполните:

```bash
hash -r
```

Либо просто откройте новую сессию терминала.
