# Руководство пользователя TuxBack

Данный документ описывает использование консольной утилиты **TuxBack**.

TuxBack позволяет:

- создавать резервные копии директорий
- восстанавливать данные из архивов
- удалять архивы резервных копий
- настраивать автоматическое резервное копирование
- просматривать состояние системы резервного копирования

---

# Создание резервной копии

Чтобы создать резервную копию директории, выполните команду:

```bash
tuxback backup <директория>
```

Пример:

```bash
tuxback backup /home/user/data
```

После выполнения команды:

- директория архивируется
- архив сжимается в формат `.tar.gz`
- файл сохраняется в директории `backups/`

Пример вывода:

```
Backup created: backup_20260315_143500.tar.gz
```

---

# Просмотр резервных копий

Чтобы увидеть список всех созданных архивов, выполните:

```bash
tuxback list
```

Пример вывода:

```
Available backups:
- backup_20260315_101200.tar.gz
- backup_20260315_143500.tar.gz
```

---

# Восстановление резервной копии

Для восстановления данных из архива используется команда:

```bash
tuxback restore <архив> <директория_восстановления>
```

Пример:

```bash
tuxback restore backup_20260315_143500.tar.gz ./restore
```

После выполнения команды архив будет распакован в указанную директорию.

---

# Удаление резервной копии

Чтобы удалить архив резервной копии, выполните:

```bash
tuxback delete <архив>
```

Пример:

```bash
tuxback delete backup_20260315_143500.tar.gz
```

Если указанный архив не существует, программа выведет сообщение об ошибке.

---

# Состояние системы

Команда позволяет получить общую информацию о состоянии системы резервного копирования:

```bash
tuxback status
```

Пример вывода:

```
TuxBack status:
- Version: 1.0.0
- Total backups: 2
- Total schedules: 1
- Enabled schedules: 1
```

Команда отображает:

- версию программы
- количество существующих архивов
- количество задач планировщика
- количество активных задач

---

# Планировщик резервного копирования

TuxBack содержит встроенный планировщик для автоматического создания резервных копий.

Планировщик автоматически запускается системой через **systemd timer**.

---

# Добавление задачи резервного копирования

Чтобы добавить задачу автоматического резервного копирования, выполните:

```bash
tuxback schedule-add <директория> <интервал_в_минутах>
```

Пример:

```bash
tuxback schedule-add /home/user/data 10
```

В данном примере резервная копия будет создаваться каждые **10 минут**.

---

# Просмотр задач планировщика

Чтобы увидеть все настроенные задачи резервного копирования:

```bash
tuxback schedule-list
```

Пример вывода:

```
Scheduled backup tasks:
- ID: 1, Source: /home/user/data, Interval: 10 min, Enabled: True
```

---

# Удаление задачи планировщика

Чтобы удалить задачу резервного копирования, выполните:

```bash
tuxback schedule-delete <id>
```

Пример:

```bash
tuxback schedule-delete 1
```

---

# Ручной запуск планировщика

Планировщик можно запустить вручную:

```bash
tuxback run-scheduler
```

Обычно эта команда запускается автоматически через **systemd timer**, но может использоваться для тестирования.

---

# Журнал работы

TuxBack ведёт журнал работы приложения.

Логи записываются в файл:

```
tuxback.log
```

Пример записи в журнале:

```
2026-03-15 10:45:21 | INFO | cli | CLI command started: backup
```

Логи позволяют отслеживать:

- создание резервных копий
- работу планировщика
- ошибки системы
